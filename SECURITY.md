# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | Yes |

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub issues.**

### Preferred: GitHub Security Advisories

1. Navigate to the repository's **Security** tab
2. Click **"Report a vulnerability"**
3. Fill out the advisory form with details
4. Submit the report

### Alternative: Email

Email security concerns to: **security@organvm.dev**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact

## Response Timeline

| Timeline | Action |
|----------|--------|
| Within 48 hours | Acknowledgment |
| Within 7 days | Assessment and severity classification |
| Within 30 days | Fix deployed (critical/high severity) |

## Security Best Practices

- Never commit secrets, API keys, or credentials
- Use environment variables for sensitive configuration
- Keep dependencies updated
- Follow secure coding practices per [CONTRIBUTING.md](CONTRIBUTING.md)
